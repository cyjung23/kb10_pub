# 의사결정 로그 DEC-001 ~ DEC-008

프로젝트 kickoff 이전 논의에서 확정된 핵심 결정 기록. 모든 후속 작업의 단일 기준(single source of truth).

---

## DEC-001 — 도메인 확정
- 결정: `kbeauty10.com` 을 메인 도메인으로 확정 (가비아 구매 완료)
- 근거: "kbeauty" 키워드 보존 + "10" 리스티클 컨셉 반영. `10kbeauty.com`(10k=10000 오인 위험), `kbeautyten.com`(철자형)보다 SEO·시각적 주목도 우위
- 방어: 필요 시 `kbeautyten.com`, `10kbeauty.com` 추가 확보 후 리다이렉트

## DEC-002 — 서비스 주제
- 결정: K-beauty 미용의료 정보 큐레이션 (피부과 중심)
- 근거: 소규모 팀 자원 한계상 광범위 관광 주제(10가지 모든 주제) 대신 토픽 authority 확보에 유리. 피부과가 외국인 환자 성장률 최상위(+86.2%)

## DEC-003 — 핵심 타깃
- 결정: 고객군 B(관광 중 피부과 시술 관광객)를 주축. 고객군 A(성형·수술 목적)는 상단 콘텐츠로 보조
- 근거: B는 시장 규모 크고 성장 빠름, 짧은 다운타임, 웹 검색·비교 행동이 사이트 모델과 일치. A는 에이전시 장악·신뢰부담 큼

## DEC-004 — 언어 전략
- 결정: English 기본(defaultLocale=en) + 한국어. 하위폴더(/en/, /ko/) + hreflang. 기계번역 금지
- 근거: 얇은 다국어 페이지 회피, 언어당 페이지 품질 우선. 루트는 en 콘텐츠 노출 + x-default, 강제 리다이렉트 없이 언어 전환 배너 제공
- SCP 교훈: defaultLocale 변경(SEO-009 이슈) 회피 위해 처음부터 en 기본으로 시작

## DEC-005 — 제목 규칙
- 결정: `10 [시술] Clinics in [지역], Seoul (2026 Guide)` 스타일 채택
- 규칙: 연도(2026)는 제목·본문·meta에만 넣고 URL에는 넣지 않음. 연도 갱신 시 콘텐츠(가격·목록)도 함께 갱신
- 근거: 정보성/가이드 표현으로 "Best/Top" 광고성 표현 회피(의료법), freshness 신호 확보

## DEC-006 — URL 구조
- 결정: 루트 아래 평면(플랫) 구조. 계층 폴더 미사용
- 조합 페이지: 시술-지역 순 (`/rejuran-gangnam/`, `/botox-myeongdong/`)
- 단독 시술 페이지: `/rejuran/`, `/botox/`
- 증상 페이지: `/darkspots/`, `/acne-scars/`
- 병원 상세: `/clinic/[clinic-name]/`
- 색인(뼈대) 페이지: `/treatments/`, `/areas/`, `/symptoms/` — 링크망 형성용
- 근거: 신규·소규모 사이트에 평면 구조 유리. 단 내부 링크망 필수(고립 페이지 방지)

## DEC-007 — 법적 기준선 (SCP 승계)
- 결정: 아래 4원칙 준수
  1. 병원으로부터 금전 미수취 (무대가)
  2. 예약·상담·송객 기능 없음 (무중계)
  3. 순위·비교·과장·전후사진·후기 배제 (중립 표현)
  4. SCP DB 원문 반드시 리라이팅 (중복 콘텐츠 회피)
- 근거: 의료법 제56조(광고주체·외국인유치 제한, 유인·과장·후기노출 금지) 리스크 회피. 무대가+무중계가 핵심 방어선
- 미결: 병원 카드 가격표기 수준(지역평균 vs 개별) — 원장 의견 대기. 외국인 타깃 특성상 의료광고 전문 변호사 자문 권고

## DEC-009 — 루트 언어 자동감지 비활성화
- 결정: next-intl `localeDetection: false` 설정. 루트(/) 접속 시 브라우저 언어와 무관하게 항상 defaultLocale(en)로 리다이렉트
- 근거: DEC-004의 "en 기본 + x-default" 방침 일치. Google 권장(자동감지 강제 리다이렉트 회피). 주 타깃이 영어권 관광객
- 보충: 검색 유입은 hreflang 걸린 개별 언어 페이지(/en/, /ko/)가 직접 노출되므로 루트 리다이렉트 설정과 무관. 한국인 한글 검색 시 /ko 페이지 정상 노출됨
- 후속 필수: 각 페이지에 hreflang 태그 정확히 삽입 (다국어 SEO 실제 엔진)
